# Docs Directory

## Purpose

The `docs/` directory is the source for project-level implementation approaches.
Use these files to keep conventions explicit, repeatable, and consistent across routes, APIs, loaders, forms, db definition etc.

## When To Use These Docs

- Starting a new feature and choosing project patterns
- Implementing flows that already have a documented convention
- Reviewing or refactoring code to align with established structure
- Resolving ambiguity before introducing a new pattern

## How To Keep It Useful

- Add focused docs per concern (auth, server actions, queries, UI patterns)
- Prefer practical rules and examples over long theory
- Update docs when conventions evolve
- Link related docs from `AGENTS.md` so guidance is discoverable
